Test API Server for the eat@home project

Use the following endpoints to access the fake data:

GET

Companies

  • Specific Company's Review: /api/comp/review/1
  • Specific Company's Rating: /api/comp/rating/3

Users

  • Users: /api/users
  • Specific User: /api/users/1
  • Specific User's Ratings: /api/users/ratings/1
  • Specific User's Reviews: /api/users/reviews/1
  • Specific User's Saves: /api/saves/1

Other

  • All Saves: /api/saves
  • All Reviews: /api/reviews
  • All Ratings: /api/ratings


POST

Adding a User:

POST, with body:

  • /api/users
  • displayName (string)
  • userName (string)
  • email (string)

Adding a Save:

POST, with body:

  • /api/saves
  • user_id (integer)
  • company_id (integer)

Adding a Review:

POST, with body (note: content of 'reviews' should be text of the review):

  • /api/reviews
  • review (string)
  • user_id (integer)
  • company_id (integer)

Adding a Rating:

POST, with body:

  • /api/ratings
  • user_id (string)
  • company_id (integer)
  • rating (integer)


PUT

Updating a User:

PUT, with body:

  • /api/users/:id
  • displayName (string)
  • userName (string)
  • NOTE: The database currently has firstName and lastName, rather than username.

Updating a Review:

PUT, with body:

  • /api/reviews/:id
  • reviews (string)

Updating a Rating:

PUT, with body:

  • /api/ratings/:id
  • rating (integer)

DELETE

Deleting a User:

DELETE:

  • /api/users/:id

Deleting a Save:

DELETE:

  • /api/saves/:id

Deleting a Review:

DELETE:

  • /api/reviews/:id

Deleting a Rating:

DELETE:

  • /api/ratings/:id
